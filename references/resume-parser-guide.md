# 简历解析指南 (Resume Parser Guide)

## 概述

本文档指导面试官技能如何有效解析和利用候选人简历信息来定制面试策略。

---

## 1. 简历信息提取框架

### 1.1 结构化提取模板

从简历中提取以下信息，整理为内部参考：

```yaml
candidate_profile:
  education:
    - degree: ""        # 学位
      school: ""        # 学校
      major: ""         # 专业
      year: ""          # 毕业年份
      gpa: ""           # GPA（如有）
      highlights: []    # 学术亮点（论文、竞赛等）

  experience:
    total_years: 0      # 总工作年限
    positions:
      - company: ""     # 公司
        title: ""       # 职位
        duration: ""    # 时间段
        tech_stack: []  # 使用的技术栈
        key_projects: []  # 核心项目
        impact: ""      # 量化成果

  technical_skills:
    languages: []       # 编程语言
    frameworks: []      # 框架
    tools: []          # 工具
    domains: []        # 领域知识
    certifications: [] # 认证

  highlights:           # 亮点
    publications: []    # 论文发表
    open_source: []     # 开源贡献
    awards: []          # 获奖
    patents: []         # 专利
    talks: []           # 演讲/分享

  inferred_strengths: [] # 推断的优势领域
  potential_gaps: []     # 可能的薄弱点
  experience_level: ""   # 推断的经验层级
```

### 1.2 信息推断规则

**经验层级推断：**
- 0-2年 + 非管理角色 → 初级
- 3-5年 + 独立负责项目 → 中级
- 5-8年 + 架构决策经验 → 高级
- 8年+ + 跨团队影响力 → Staff/Principal

**优势领域推断：**
- 简历中用大篇幅描述的项目 → 核心能力区
- 有量化成果的经验 → 强项（有产出证明）
- 持续使用2年以上的技术 → 深度掌握

**薄弱点推断（谨慎标记）：**
- 简历中提及但未详细描述的技术 → 可能只是了解层面
- 工作经历中的空白期 → 需温和探查（不直接质问）
- 技术栈转换频繁 → 可能每项都不够深入
- 缺少目标方向的核心技能 → 需要考察转化能力

## 2. 基于简历的面试策略定制

### 2.1 项目深挖策略

**选择深挖的项目标准：**
1. 与面试方向最相关的项目
2. 候选人担任核心角色的项目
3. 有量化成果但描述模糊的项目（可能是包装）
4. 技术复杂度最高的项目

**深挖问题模板：**
- "我看到你在 {项目} 中使用了 {技术}，能详细讲讲具体是怎么做的吗？"
- "这个项目中最大的技术挑战是什么？你是怎么解决的？"
- "你提到这个项目提升了 {指标} XX%，能讲讲是怎么衡量的吗？"
- "如果让你重新做这个项目，你会做什么不同的决策？"

### 2.2 技术栈匹配策略

**当候选人技术栈与目标方向高度匹配时：**
- 考察深度：问底层原理和高级用法
- 考察广度：延伸到生态和相关技术
- 考察判断力：技术选型决策和权衡

**当候选人技术栈与目标方向部分匹配时：**
- 考察迁移能力：相似概念在不同技术中的理解
- 考察学习能力：对新技术的了解程度
- 降低工具细节要求，提高原理理解要求

**当候选人是转方向时：**
- 关注可迁移的底层能力
- 考察学习动机和路径
- 适当降低领域专深要求
- 提高问题解决能力和学习能力的权重

### 2.3 经验亮点验证

对简历中的突出成就，使用以下验证策略：

**论文/研究：**
- 请简要介绍你论文的核心贡献
- 这个方法的局限性是什么？
- 后续有人基于你的工作做了什么扩展吗？

**开源贡献：**
- 你在这个项目中主要负责什么？
- 印象最深的一个PR是什么？
- 如何处理社区反馈和code review？

**量化成果：**
- 这个 XX% 的提升是怎么衡量的？
- 对比基准是什么？
- 这个改进对业务的实际影响是什么？

## 3. 简历格式处理

### 3.1 支持的格式

| 格式 | 处理方式 |
|-----|---------|
| PDF | 使用 Read 工具读取（或PDF技能） |
| DOCX | 使用 Read 工具读取文本内容 |
| 纯文本 | 直接在对话中提供 |
| 图片 | 使用 Read 工具查看（OCR） |
| LinkedIn URL | 使用 WebSearch/WebFetch 获取公开信息 |

### 3.2 隐私注意事项

- 不保存候选人的个人联系信息（电话、邮箱、地址）
- 简历分析结果仅用于当次面试
- 不对候选人的个人背景做假设或评判
- 聚焦于专业能力和经验，不涉及个人生活
